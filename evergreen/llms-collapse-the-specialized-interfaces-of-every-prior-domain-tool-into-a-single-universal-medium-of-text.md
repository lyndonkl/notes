# LLMs collapse the specialized interfaces of every prior domain tool into a single universal medium of text

Source: [[thought-session-what-the-g-in-agi-actually-means|Thought Session: What the G in AGI Actually Means]]

Working in any domain has historically required mastering the interface of that domain's tools. PCR machines and autoclaves for biochemistry, PyTorch and Jupyter for machine learning, Excel for finance, statistical software for data analysis — each tool came with its own interaction model, its own conventions, and its own documentation that had to be internalized before the tool could become useful.

The LLM collapses this entire layer. The interface to domain knowledge is now natural language text — the same medium used to query, refine, instruct, and extend. A user no longer needs to know how to pipette, how to call PyTorch's training loop, how to write VLOOKUP, or how to compose a SQL query in order to enter a domain. They need to know how to construct a context that scopes the model toward useful output for that domain.

This interface collapse is one half of what makes the LLM a historically distinctive tool. The other half is that the knowledge of each domain is already present in the model. Either alone would be unremarkable: a universal interface without knowledge is a shell prompt; broad knowledge behind specialized interfaces is just an encyclopedia. Their combination is the historical break.

**Open questions:**
- Some interfaces are not collapsible into text — embodied tools, real-time motor control, hardware operation. Where does the interface-collapse claim stop being a description of generality and start being a description of "things that can be reduced to text"?
- Is the universal interface really "text," or specifically "natural language"? The distinction matters because it implies what the next generalization (e.g., video, action, multimodal embodiment) might look like.
