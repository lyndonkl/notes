# Language models enable a shift from reactive to context-aware initiative in machine interfaces

Source: [[thought-session-what-the-g-in-agi-actually-means|Thought Session: What the G in AGI Actually Means]]

The dominant model of human-machine interaction has been reactive: the user issues a command, the machine responds. Click a button, get a result. Type a query, get an answer. The user holds the initiative; the machine waits. This pattern has been universal because previous machines lacked the capacity to maintain rolling context about a user's life or to infer what action would be useful at a given moment.

Language models change this. A system that maintains context about what a user cooks, what a user spends, what a user is shopping for, and what's currently in their fridge can infer that, when the user is driving past a grocery store, a particular item is worth picking up. The system can initiate this nudge without being asked. The interface shifts from "user commands, machine responds" to "machine maintains context, infers usefulness, and initiates."

This is not "the AI becomes proactive" in the sense of having its own purposes. The system's initiative is still operationalizing the user's purposes (eating dinner, finding good deals) at a different point in the loop than command-response interaction allows. The framing matters because the new interface introduces obligations the old interface did not — privacy, consent, attention, the right to not be initiated-on. The system can only have initiative in proportion to what it observes about the user, and observation is not free.

**Open questions:**
- What design patterns prevent context-aware initiative from becoming pestering? The grocery example is useful; the same mechanism applied without restraint is intolerable.
- How does the user maintain effective evaluation when the system is initiating across many domains simultaneously? The human-as-terminal-evaluator role gets harder when many threads are running in parallel.
